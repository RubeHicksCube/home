Have you ever wanted to make a website, but everywhere you look you see that you need to pay for hosting, pay for the domain, pay for the application that runs your site, pay for the plugins in the site... You know, that everything costs an arm and a leg just to share a welcome page?

I'm going to show you that you really don't need to pay for all of that. That all you need is a domain name if you want, but it's not required.

What's Needed?

  1. A GitHub account
  2. A Netlify account
  3. Microsoft Word or Google Docs (something that exports to HTML)
  4. Visual Studio Code (optional for customization)

Getting Started

When you go to GitHub, create a new repository.

  1. Name the Project
  2. Make it public (to access online)
  3. Create Repository

Create a Netlify Account

When you create the Netlify account, link it to your GitHub account. This will enable it to automatically refresh with any changes you make to your repository, and publish them as a webpage with your custom domain. Essentially making your github a free hosting server and Netlify as a DNS server.

Creating the Site

Once your site is set up, you'll need some files.

To test, you can save any web page as an html document and upload that to your GitHub repository.

There are copyright laws with doing this, so remember to only share what you own!

My recommendation is to create a very basic outline of a webpage on Google Docs and save it as an HTML document, and then upload that HTML document to GitHub.

And just like that you have a webpage! Absolutely free, with no coding experience.
